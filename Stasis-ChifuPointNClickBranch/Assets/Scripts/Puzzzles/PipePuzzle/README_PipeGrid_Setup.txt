═══════════════════════════════════════════════════════════════
  PIPE GRID — КАК СОБРАТЬ В UNITY (ПОШАГОВО)
═══════════════════════════════════════════════════════════════

Идея: трубы уже стоят на сетке. Клик по ячейке — поворот на 90°.
Нужно соединить путь от точки A до точки B. Победа — событие On Puzzle Solved.

Ниже — пример на сцене ColleagueCabinet и окне как у карты (Puzzle).
Можно повторить в любой сцене с Canvas и EventSystem.

───────────────────────────────────────────────────────────────
ШАГ 0. Что должно быть в сцене
───────────────────────────────────────────────────────────────
• Canvas (у вас под ===UI=== → Canvas)
• На Canvas — компонент Graphic Raycaster (обычно уже есть)
• В сцене — EventSystem (у вас часто под ===Managers===)
Если чего-то нет: GameObject → UI → Canvas (создаст Canvas + EventSystem).

───────────────────────────────────────────────────────────────
ШАГ 1. Окно головоломки (панель)
───────────────────────────────────────────────────────────────
Вариант А — как у карты:
• Перетащите префаб Assets/Prefabs/Puzzle внутрь Canvas
• Переименуйте в PuzzlePipeGrid (чтобы не путать с другими)
• Разверните: внутри будут PuzzleBackground и PuzzlePanel

Вариант Б — только панель без затемнения:
• ПКМ по Canvas → UI → Panel
• Переименуйте в PipeGridPanel

Дальше в инструкции считаем, что контейнер для поля — PuzzlePanel
(или ваш Panel).

───────────────────────────────────────────────────────────────
ШАГ 2. Контейнер сетки (удобно двигать всё поле целиком)
───────────────────────────────────────────────────────────────
• ПКМ по PuzzlePanel → Create Empty
• Переименуйте в PipeGridField
• Выберите PipeGridField → Add Component:
    — PipeGridPuzzle
    — PipeGridGenerator

ВАЖНО: удалите лишнее под PipeGridField
• Если там десятки копий Cell_0_0 — выделите их все и Delete (генератор создаст заново).
• Объект PipePuzzle (старый drag-префаб) внутри PuzzlePanel — можно УДАЛИТЬ,
  если вы играете только в сетку-поворот. Иначе оставьте.

───────────────────────────────────────────────────────────────
ШАГ 2Б. Иерархия — правильно ли (как у вас на скрине)
───────────────────────────────────────────────────────────────
Правильная схема:

  Canvas
    PuzzlePipeGrid          ← окно (префаб Puzzle)
      PuzzleBackground
      PuzzlePanel
        CloseButton
        PipeGridField       ← ПУСТОЙ контейнер + скрипты
          (после генерации) Cell_0_0 … Cell_7_4

На объекте PipeGridField должны висеть:
• Rect Transform
• PipeGridPuzzle   (логика A→B, On Puzzle Solved)
• PipeGridGenerator (авто-создание клеток)
• Grid Layout Group — появится САМ после первой генерации

Не обязательно держать рядом старый «Puzzle» и «PipePuzzle» на Canvas —
достаточно одного PuzzlePipeGrid для этой головоломки.

───────────────────────────────────────────────────────────────
ШАГ 3. Сетка 8×5 одной командой (PipeGridGenerator)
───────────────────────────────────────────────────────────────
1) Выберите PipeGridField в Hierarchy.

2) В Inspector найдите компонент Pipe Grid Generator.

3) Выставьте:
   • Columns = 8
   • Rows = 5
   • Cell Size — размер клетки (например 90 × 90)
   • Pipe Graphic Scale — насколько увеличить только рисунок трубы (клетка на сетке
     того же размера; без этого линии могли оставаться мелкими внутри большой клетки)
   • Spacing — зазор между клетками
   • Cell Sprite — любой спрайт трубы из PipePuzzle (или один на всех)
   • Default Shape — Straight / Corner / Tee / Cross / Empty (пустая, без прохода)
   • Randomize Start Rotation — галочка, если нужен случайный стартовый поворот
   • Pipe Grid Puzzle — перетащите сюда тот же объект PipeGridField
     (если пусто, скрипт сам найдёт PipeGridPuzzle на этом же GameObject)

4) ПКМ по заголовку компонента «Pipe Grid Generator»
   (маленький шестерёнка/три точки справа от имени компонента — в Unity это меню)
   → выберите пункт:
   **«Удалить детей и сгенерировать сетку»**

   Либо: ПКМ по имени компонента в Inspector → тот же пункт Context Menu.

5) Под PipeGridField появятся Cell_0_0 … Cell_7_4 (40 штук).
   В PipeGridPuzzle автоматически выставятся:
   Start Cell (0, 0), End Cell (7, 4).

6) Сохраните сцену (Ctrl+S).

При необходимости снова запустите пункт меню — старые дети удалятся и сетка
пересоздастся (настройки Shape/спрайт возьмутся из генератора).

Север в коде = увеличение Y (клетка выше на экране = y+1).
Нижний ряд y=0, верхний y=4.

───────────────────────────────────────────────────────────────
ШАГ 3А. Ручная правка отдельных клеток (по желанию)
───────────────────────────────────────────────────────────────
После генерации можно выбрать любую Cell_x_y и сменить Image / Shape / Rotation
в Inspector — Grid Position трогать не нужно (уже совпадает с именем).

───────────────────────────────────────────────────────────────
ШАГ 3Б. Старая инструкция — одна ячейка вручную (если без генератора)
───────────────────────────────────────────────────────────────
ШАГ 3-old. Одна ячейка (шаблон)
───────────────────────────────────────────────────────────────
• ПКМ по PipeGridField → UI → Image
• Переименуйте, например, в Cell_0_0

На объекте Cell_0_0:
1) Image:
   • Source Image — ваш спрайт трубы (из Assets/Sprites/PipePuzzle/…)
   • Raycast Target — ВКЛ (галочка), иначе клик не поймается

2) Add Component → PipeGridCell

3) PipeGridCell:
   • Grid Position: X = 0, Y = 0  (должно совпадать с именем/ролью клетки)
   • Shape:
       Straight — прямая (в базе W–E)
       Corner   — угол (в базе S+W, как ваш тайл)
       Tee      — тройник
       Cross    — крест
   • Rotation Quarters: 0 … 3 — начальный поворот (по желанию)

Скопируйте ячейку для остальных позиций:
• Ctrl+D дублирует объект
• Перетащите копии в PipeGridField (если вылезли наружу)
• Для каждой копии:
  — задайте уникальное Grid Position (0,0) (1,0) (2,0) …
  — поставьте нужный Shape и спрайт
  — при необходимости подправьте RectTransform (позиция в сетке)

Пример сетки 3×3 (координаты):
  (0,2) (1,2) (2,2)
  (0,1) (1,1) (2,1)
  (0,0) (1,0) (2,0)

Start (0,0) и End (2,2) — диагональ; вы можете поставить A и B как угодно,
главное — те же числа в PipeGridPuzzle и в ячейках.

───────────────────────────────────────────────────────────────
ШАГ 4. Размер клеток и центрирование
───────────────────────────────────────────────────────────────
• Если использовали PipeGridGenerator — размер и отступы в нём (Cell Size, Spacing).
  После генерации Grid Layout Group выставляет Child Alignment = Middle Center —
  сетка остаётся по центру, если контейнер (PipeGridField) растянут на родителя:
  Anchors Min (0,0) Max (1,1), Left/Right/Top/Bottom = 0, Size Delta 0.
• Если собирали вручную — Width/Height у Rect Transform каждой Image и/или Grid Layout Group.

───────────────────────────────────────────────────────────────
ШАГ 5. Проверка в Play
───────────────────────────────────────────────────────────────
• Сохраните сцену (Ctrl+S)
• Play
• Кликайте по трубам — они должны крутиться на 90°
• Когда получится непрерывный путь от Start до End, сработает On Puzzle Solved

Если клик не работает:
• Raycast Target на Image включён
• Graphic Raycaster на Canvas есть
• EventSystem в сцене есть
• Нет другого UI поверх, перехватывающего клики

Если сетка «уезжает» в угол и остаётся мелкой:
• У КОРНЯ окна (префаб Puzzle / PuzzlePipeGrid) RectTransform должен быть
  на весь Canvas: Anchors (0,0)–(1,1), Size Delta (0,0). Старый вариант
  100×100 по центру ломает Layout и GridLayoutGroup — дети «растягиваются»
  от крошечного родителя.

───────────────────────────────────────────────────────────────
ШАГ 6. Окно скрыто до открытия (как у карты)
───────────────────────────────────────────────────────────────
• Выключите галочку Active у корня PuzzlePipeGrid (весь Puzzle)
• Добавьте на сцену пустой объект с PipePuzzleTestOpener
• В поле Pipe Puzzle Window перетащите корень Puzzle (тот же, что для карты,
  или отдельный Puzzle только для сетки — как решите)
• По клавише P окно откроется — там будет и ваш PipeGridField внутри панели

Если PipeGrid в отдельном Puzzle-префабе — привяжите именно его корень
к PipePuzzleTestOpener.

───────────────────────────────────────────────────────────────
КОД (напоминание)
───────────────────────────────────────────────────────────────
• PipeGridPuzzle — только на PipeGridField (родитель ячеек)
• PipeGridCell — на каждой клетке с Image
• Победа: после каждого поворота проверяется путь от Start до End по направлениям труб
  (стык только если у обеих клеток открыты противоположные стороны). Событие — On Puzzle Solved.
• У клеток Start и End в логике должны быть открыты стороны к соседям на пути (например финиш слева
  в сетке должен иметь открытие на восток к соседу). Декоративный угол без «входа» с нужной стороны
  сделает уровень непроходимым.
• У каждой клетки в Inspector должен совпадать Shape (Straight/Corner/Tee/Cross/Empty) с тем, что нарисовано на спрайте,
  иначе логика направлений не совпадёт с картинкой.
• Empty — пустая клетка без направлений: по ней нельзя пройти, клик не крутит (поставьте спрайт «пусто» в Image).
• После победы: puzzle.LastSolvedPath — клетки пути A→B

───────────────────────────────────────────────────────────────
ЛОГИКА (сессия, события, дальнейший код)
───────────────────────────────────────────────────────────────
• PipeGridPuzzleSession — на корень окна (тот же объект, что Puzzle / PuzzlePipeGrid):
  при открытии окна (OnEnable) вызывает сброс. По умолчанию —
  ResetPuzzleToInitialLayout() (повороты как в сцене + сброс «решено»).
  Можно выключить «сброс поворотов», тогда только ResetSolvedState().
• Событие On Puzzle Solved (PipeGridPuzzle) — в Inspector привяжите звук, квест,
  закрытие окна (например вызов Puzzle.Disable на корне), другой скрипт.
• API: IsSolved, StartCell / EndCell, ResetPuzzleToInitialLayout(), RefreshCellsFromHierarchy()
• Новая логика (квесты, сохранения) обычно подписывается на onPuzzleSolved или вызывается
  из вашего класса по ссылке на PipeGridPuzzle — раскладку клеток заново делать не нужно.
